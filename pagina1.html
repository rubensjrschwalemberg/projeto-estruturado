Questionário 2



Para que serve o commit e qual a melhor forma de utilizá lo?

Commit serve para gravar alterações quando você quiser no seu repositório. Os estados dos arquivos são: untracked, unmodified, modified e staged. Untracked são arquivos que ainda não foram incluídos na última foto e não estão em stage. Unmodified são os arquivos que são clonadas pela primeira vez sem ser editados. Modified são os arquivos que foram modificados após o último commit. Então você comita as alterações e o ciclo se repete. A melhor forma de utilizá-lo é comitando cada alteração importante no arquivo.



Qual a diferença entre commit e push

push é utilizado para enviar um arquivo para um lugar específico. Por exemplo: git push origin master. Enviar os arquivos da pasta pessoal para o github. O commit grava alterações no repositório quando você quiser.



O que é tag

É utilizado para marcar pontos específicos no desenvolvimento de aplicativos ou softwares. Existem dois tipos de tags a leve e anotada. A tag leve aponta para um commit específico. A tag anotada ela tem marcações de nome, e-mail, data, mensagem. Podem ser assinadas.



O que é branch

Branch é um ponteiro móvel que aponta para um commit. Cada vez que é feito um commit o branch aponta para ele. Para identificar o commit é feito um snapschot da tela e salvo numa hash que é um código de letras e número que o identifica. É possível criar branches em lugares diferentes da master. Possui várias vantagens entre elas: alterar o arquivo sem modificar o master, fácil de criar e excluir, várias pessoas podem trabalhar ao mesmo tempo e evita conflitos.
